# Prompting with LLMs When Building AI Agents

## Why Prompting Matters

Since the release of **ChatGPT**, LLMs have evolved into much more
capable models, and they will continue to evolve. Today, there are many
models from different companies, and not every LLM will produce the same
response for the same prompt.

The quality of a response depends on several factors, including:

-   Model architecture
-   Model size
-   Training data
-   Fine-tuning and post-training
-   Prompt quality

Before learning how to write good prompts, understanding how an LLM
works is important.

------------------------------------------------------------------------

# How an LLM Works

To put it simply, **LLMs are fundamentally next-token predictors.**

A token is not always a complete word. Depending on the tokenizer, it
can be:

-   A complete word
-   Part of a word
-   Punctuation
-   Whitespace

Whether it is reasoning, code generation, summarization, or answering
questions, every response generated by an LLM is ultimately produced by
predicting the next token repeatedly until the response is complete.

Because of this, crafting a good prompt is important.

------------------------------------------------------------------------

# Temperature

One important factor during token prediction is **temperature**.

A common misconception is that increasing the temperature produces
better responses. In reality, temperature only changes the randomness of
the output.

Temperature controls how randomly the next token is selected from the
probability distribution generated by the model.

-   Lower temperature → More deterministic responses
-   Higher temperature → More diverse and creative responses

There is no single temperature value that works for every task. The
ideal value depends on what you are trying to achieve.

### Typical Temperature Values

-   **Summarization:** `0.0 – 0.3`
-   **Question Answering:** `0.1 – 0.5`
-   **Code Generation:** `0.2 – 0.7`
-   **Brainstorming:** `0.7 – 1.0`
-   **Creative Writing:** `0.8 – 1.2`

> These are only general guidelines. Different models may behave
> differently, so experimenting with temperature is always recommended.

# Prompt Categories

## 1. Direct Prompt

A direct prompt simply tells the model what to do without providing
examples or asking it to reason explicitly.

Modern LLMs are capable enough that direct prompts work surprisingly
well for many tasks.

**Advantages**

-   Uses fewer input tokens
-   Faster responses
-   Works well for straightforward tasks

**Example**

``` text
Summarize the following article into five bullet points.
```

------------------------------------------------------------------------

## 2. Zero-shot Reasoning Prompt

A zero-shot reasoning prompt encourages the model to spend more effort
reasoning before producing the final answer.

For modern reasoning models, this reasoning happens internally rather
than exposing the complete reasoning process.

These prompts may take slightly longer and can consume more reasoning
tokens, but they often improve performance on complex tasks.

**Example**

``` text
A customer paid ₹250 for three notebooks costing ₹65 each.
Reason through the problem before giving the final answer.
```

------------------------------------------------------------------------

## 3. Few-shot Prompt

Few-shot prompting provides examples of the expected input and output
format before asking the actual question.

This helps the model understand the required structure and usually
improves consistency.

Typically, few-shot prompts include:

-   One or more examples
-   Expected output
-   Clear constraints

These prompts use more input tokens because of the examples, but they
generally produce higher-quality and more consistent responses.

**Example**

``` text
Convert the following sentences into JSON.

Example

Input:
John is 25 years old.

Output:
{
  "name": "John",
  "age": 25
}

Now convert:

Alice is 30 years old.
```
