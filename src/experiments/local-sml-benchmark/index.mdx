<ExperimentOverview>

This experiment compares small language models (SLMs) running locally, with a focus on measuring and benchmarking practical metrics side by side.

The goal is to understand how different SLMs behave under the same local setup — latency, throughput, resource usage, and output quality — so the trade-offs are clear before picking a model for a real workload.

Architecture, tooling, and detailed results will be documented as the experiment progresses.

<ExperimentTimeline>

  <ObservationBlock title="Overall observation">

    - Compared gemma3:270m, qwen3:0.6b, and qwen3:1.7b.
    - Measured speed using latency and tokens/sec.
    - Measured quality using GPT-based evaluation.
    - Generated reusable benchmark/report files.

    This experiment was useful because the same benchmark flow can be reused to test any local model with different prompts. Comparing models this way helps choose the best model for a particular use case.
      - `qwen3:0.6b` has the highest tokens/sec.
      - `gemma3:270m` has the lowest average response duration.
      - `qwen3:1.7b` has the highest response quality, but it costs more in terms of response duration.

    Because my system has limited RAM and disk space, I could not test larger models in this version. In a future version, after upgrading my system, I can repeat the same benchmark with bigger models.

    **System configuration**
    - Device: MacBook Air M2
    - RAM: 8 GB
    - Storage: 256 GB SSD
