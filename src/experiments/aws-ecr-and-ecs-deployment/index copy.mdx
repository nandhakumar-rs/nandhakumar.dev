<ExperimentOverview>

I am experimenting with building a complete LLM chat application using React and FastAPI.

The purpose is to understand how an LLM application moves from a simple local prototype to a containerized application deployed on AWS. I want to learn how to structure versioned iterations, add streaming and observability, and deploy with minimal operational overhead.

The final goal is a working chat application with conversation history, streaming responses, containerized services, and basic evaluation metrics — all documented version by version.

### Lessons learned

- Start with the simplest API endpoint before building UI complexity
- Streaming significantly improves perceived latency for chat applications
- Containerizing early makes deployment iterations faster
- Observability should be planned before production deployment

### Known limitations

- Conversation history is not yet persisted to a database
- Evaluation metrics are placeholder values
- No authentication or rate limiting implemented
