<ExperimentOverview>

This experiment explores the fundamentals of regression through house price prediction.

The plan is to start with a synthetic dataset to understand core concepts — features, targets, train/test splits, and model evaluation — then gradually move to a real-world dataset (UCI Real Estate).

Tools, architecture, and versioned progress will be documented as the experiment moves forward.

  <ExperimentVersion
    version="v9"
    title="Feature scaling"
    status="completed"
    gitBranchUrl="https://github.com/nandhakumar-rs/house-price-prediction/tree/v9/feature-scaling"
  >
    The input features have different value ranges, which can cause gradient descent to update some feature weights faster than others.

    Used standardization to bring the features to a similar scale:

    <CodeSnippet language="text">
{`scaled_value = (value - mean) / standard_deviation`}
    </CodeSnippet>

    This centers each feature around 0 with a standard deviation of approximately 1, helping gradient descent update the feature weights more evenly.

    - Created `scaler.py` to calculate the mean and standard deviation and scale the features around `0`
    - Created `train_test_uci_scaled_features.py` to split the dataset, scale the train and test features, train the model, and calculate MSE
    - Scaling values are calculated only from the training dataset to avoid test data leakage
    - Trained the model with a learning rate of `0.01` and `5000` epochs

    <CodeSnippet language="text">
{`features=house_age,convenience_stores
target=house_price_per_unit_area
train_rows=332
test_rows=82

scaling_values
house_age_mean=18.0572
house_age_std=11.3310
convenience_stores_mean=4.0211
convenience_stores_std=2.9600

epoch,house_age_weight,convenience_stores_weight,bias,train_mse
1,-0.0254,0.0776,0.3792,1598.1477
1000,-3.1119,7.9867,37.9185,120.2529
2000,-3.1123,7.9873,37.9202,120.2529
3000,-3.1123,7.9873,37.9202,120.2529
4000,-3.1123,7.9873,37.9202,120.2529
5000,-3.1123,7.9873,37.9202,120.2529

final_house_age_weight=-3.1123
final_convenience_stores_weight=7.9873
final_bias=37.9202
final_train_mse=120.2529
final_test_mse=88.5699`}
    </CodeSnippet>

    **Comparison with v8**

    <NotionTable name="house-price-scaling-comparison" />

    After scaling, the model reached almost the same result in `5000` epochs that the unscaled model reached after `20000` epochs.

    Feature scaling did not reduce the final MSE by a large amount, but it helped gradient descent reach the result faster.

    Since the features are scaled, each weight now represents the price change for one standard deviation increase in that feature.

    In the next version, I will scale and include all the remaining features in the regression model.

  </ExperimentVersion>

    Train MSE and test MSE improved compared to v7 single-feature training.

    <CodeSnippet language="text">
{`final_house_age_weight=-0.1584
final_convenience_stores_weight=3.0769
final_bias=27.5312

    </CodeSnippet>

    **20000 epochs**

    At `20000` epochs, the model improved further.

    The weights were still changing, but with smaller differences.

    Stopped at `20000` epochs because the model was getting close to convergence.

    <NotionTable name="house-price-multi-feature-comparison" />

    - Final v8 result is better than v7.
    - Adding `house_age` helped, but only after training for more epochs.
    - This shows that multiple-feature training may need more time to converge.

    In the next version, feature scaling can help gradient descent converge faster and more cleanly.
