<ExperimentOverview>

This experiment explores the fundamentals of regression through house price prediction.

The plan is to start with a synthetic dataset to understand core concepts — features, targets, train/test splits, and model evaluation — then gradually move to a real-world dataset (UCI Real Estate).

Tools, architecture, and versioned progress will be documented as the experiment moves forward.

  <ExperimentVersion
    version="v8"
    title="Training model with multiple features"
    status="completed"
    gitBranchUrl="https://github.com/nandhakumar-rs/house-price-prediction/tree/v8/train-test-mulitple-feature-dataset"
  >
    - Chose one more feature, `house_age`, in addition to `convenience_stores` as input features, with `house_price_per_unit_area` as the prediction target.

    - Updated `model.py` to add a new method for prediction with multiple features.

    - Updated `data_split.py` to add a new method to split train and test data from a multiple-feature dataset.

    - Created `training_multiple.py` to calculate gradients and update weights for multiple features and bias.

    - Created `train_test_uci_multiple_features.py` to run training and testing with:
      - learning rate: `0.001`
      - epochs tested: `5000`, `10000`, and `20000`

    **5000 epochs**

    At `5000` epochs, train MSE and test MSE became worse compared to single-feature training.

    `house_age_weight` was still positive, which did not look right because normally as house age increases, price may decrease.

    Both feature weights were still changing clearly, so the model had not fully converged yet.

    <CodeSnippet language="text">
{`final_house_age_weight=0.0334
final_convenience_stores_weight=3.7010
final_bias=20.1136
final_train_mse=148.1289
final_test_mse=114.6253`}
    </CodeSnippet>

    **10000 epochs**

    At `10000` epochs, the result improved.

    `house_age_weight` became negative, which makes more sense.

    Train MSE and test MSE improved compared to v7 single-feature training.

    <CodeSnippet language="text">
{`final_house_age_weight=-0.1584
final_convenience_stores_weight=3.0769
final_bias=27.5312
final_train_mse=124.2255
final_test_mse=92.2336`}
    </CodeSnippet>

    **20000 epochs**

    At `20000` epochs, the model improved further.

    The weights were still changing, but with smaller differences.

    Stopped at `20000` epochs because the model was getting close to convergence.

    <NotionTable name="house-price-multi-feature-comparison" />

    - Final v8 result is better than v7.
    - Adding `house_age` helped, but only after training for more epochs.
    - This shows that multiple-feature training may need more time to converge.

    In the next version, feature scaling can help gradient descent converge faster and more cleanly.
